# Pynomaly Design System

Welcome to the **Pynomaly Design System** - a comprehensive, production-ready component library for building accessible and performant anomaly detection interfaces.

## 🎯 **Overview**

This Storybook instance provides interactive documentation for all UI components, design patterns, and accessibility guidelines used in the Pynomaly platform. Each component is thoroughly documented with:

- **Interactive Examples** - Live component demonstrations
- **Accessibility Guidelines** - WCAG 2.1 AA compliance documentation  
- **Code Examples** - Ready-to-use HTML and CSS
- **Design Tokens** - Color palettes, typography, spacing
- **Best Practices** - Implementation recommendations

## 🎨 **Design Principles**

### Accessibility First
Every component is designed and tested for **WCAG 2.1 AA compliance**:
- 4.5:1 color contrast ratio for normal text
- Keyboard navigation support
- Screen reader compatibility
- Clear focus indicators
- Semantic HTML structure

### Performance Optimized
Components are built for **production performance**:
- Minimal CSS and JavaScript footprint
- Progressive enhancement approach
- Efficient rendering and re-rendering
- Mobile-first responsive design
- Core Web Vitals optimization

### Developer Experience
Tools and documentation designed for **efficient development**:
- Clear API documentation
- Copy-paste code examples
- Comprehensive testing scenarios
- Cross-browser compatibility notes
- Integration with popular frameworks
